# VibeLink Mobile App

React Native app -- your interface to Claude Code from your phone. Browse projects, start sessions, chat with streaming responses, and interact with dynamic UI components that Claude pushes to your screen.

## Screen Flow

```
App Launch
  |
  +-> First time: Setup Screen (enter Bridge URL, saved permanently)
  |
  +-> Session List (home screen)
        |
        +-> "New Chat" --> Project Picker --> Chat Screen
        |
        +-> Tap existing session --> Chat Screen
                                       |
                                       +-- CLI Tab (raw event stream)
                                       +-- GUI Tab (rich markdown + components)
                                       +-- Dynamic Tabs (created by Claude via MCP)
```

### Session List

Home screen. Shows active and recent sessions with project name, last message preview, and timestamp. Long-press to delete a session. Connection badge in the header shows Bridge connectivity status (green/red dot).

### Project Picker

Calls `GET /projects` on the Bridge. Shows directories found with `.git` or `CLAUDE.md`, with badges for each. Search/filter bar at the top. Tap a project to create a new session (`POST /sessions`) and navigate to the Chat Screen.

### Chat Screen

Tabbed interface. CLI tab shows the raw NDJSON event stream. GUI tab renders Claude's responses as rich markdown with inline tool activity and dynamic UI components. Additional tabs can be created by Claude via the `create_tab` MCP tool. All tabs share the same WebSocket connection and input bar.

## State Management

Three Zustand stores, each focused on a single concern:

```
SessionStore
  - sessions: Map<id, Session>
  - activeSessionId: string | null
  - setSessions, addSession, removeSession, setActiveSession

MessageStore (keyed per session)
  - events: Map<sessionId, unknown[]>        -- raw NDJSON events (CLI tab)
  - messages: Map<sessionId, unknown[]>      -- parsed messages (GUI tab)
  - components: Map<sessionId, Map<id, component>>  -- dynamic UI components
  - tabs: Map<sessionId, tab[]>              -- dynamic tabs
  - isStreaming: Map<sessionId, boolean>
  - appendEvent, appendMessage, setComponent, setTabs, setStreaming, clearSession

ConnectionStore
  - bridgeUrl: string
  - authToken: string
  - isConnected: boolean
  - setBridgeUrl, setAuthToken, setConnected
```

## Building

### Development

```bash
npm install
npx expo start                    # Expo dev server (use Expo Go or dev client)
```

### Android APK

```bash
npx expo prebuild --platform android
cd android
./gradlew assembleRelease
```

APK output: `android/app/build/outputs/apk/release/app-release.apk`

The `/android` and `/ios` directories are gitignored -- they are generated by `expo prebuild`.

### iOS

Requires a Mac with Xcode and an Apple Developer account.

```bash
npx expo prebuild --platform ios
npx expo run:ios --device --configuration Release
```
